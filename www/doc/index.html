Fledger

Fast and Fun Ledger, or Fledger, is distinctive in the following points with most of the current blockchains:

  1. Bring back the fun - remember when bitcoin was there to change the world? Now it's just about HODLing
  1. Easy setup - no need to configure your node or your own network - just start the docker
  1. Fast rewards - once your node is running, get direct (small) rewards within minutes
  1. Actually useful - sell and buy memory, HD, CPU, network bandwidth

From a technological point of view, allow the following goodies described by the Chainiac, Byzcoin, and OmniLedger papers:

  1. Every block is final - by using a fast BFT algorithm, no forks can happen
  1. Multi-level ForwardLinks - small proofs: no need to look at all block headers to confirm a transaction
  1. Sharding - cut the processing power of the nodes in little units so that more nodes means more speed
  1. Heavy Transaction / Small Global State - instead of having a huge global state, the client needs to keep track of its own memory
  1. Forgetting stuff over time

What Fledger does NOT want to be:

  1. DeFi related - the goal of Fledger is not to allow financial transactions fast and securely
  1. Investment product - FFmana will be sold at a stable price for those wanting to profit from the resources of the network

And what you need to know:

  1. The author of Fledger dreams of working only on this project - so he allows himself to influence the FFmana economy by selling them at a fixed price to pay the development of Fledger
  1. The standard distribution of Fledger always includes at least one node of the main developer in the library chain

Is Fledger for you?

In its current state, Fledger is for you if you:

  • have a running server connected 24/7 on the internet
  • want to bring out the anarchist in you - but still want consensus on some issues
  • love trying out new stuff
  • know some rust / typescript

Minimum Viable Product

To show how good a blockchain can be, the MVP of Fledger will implement the following:

  • Single chain allowing
    • CoSi BFT signing of new blocks
    • nodes to join and quit
    • every node receives a fixed mana with every block
    • mana can be transferred from one account to another
  • Library written in Rust
    • keeping central data structures (blocks, accounts, transaction, node, ...)
    • cryptographic primitives
    • compiled to WASM for web-node
  • Nodes
    • Rust networking using webrtc with CLI status output and/or webserver for interaction with the node
    • Web node using webrtc

The roadmap for further development will include (in any order)

  • Library chain
    • holding node list
    • offering mining transactions
    • including highest-rated nodes
    • simple smart contracts
  • Shards
    • allow sharding of the nodes
    • distribute accounts to the different shards
  • Node code
    • integrated update of core code
    • web-interface to interact
  • Client
    • cli to interact with the chain
